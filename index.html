Getting Started with MQTT

Mosquitto
Mosquitto is a free and light-weight MQTT broker that can be used on any platform to mediate messages between client. To get started some tools are required. The first step is to install a local broker application. In order to do so, please perform the following steps:
  • First install the OpenSSL implementation for Windows here.
    • Install the Client and navigate to the installation folder.
    • Access the 'bin' folder and keep the folder open, e.g. 'C:\OpenSSL-Win32\bin'.
  • Download the Mosquitto Client for Windows here.
    • Install the Mosquitto application.
    • Navigate to the installation folder, e.g. 'C:\Program Files (x86)\mosquitto'.
    • Copy all DLL files from the 'bin' folder in OpenSLL to the Mosquitto installation directory.
  • Finally we need to copy some additional mandatory DLL files to make mosquitto run on Windows. All files have to be saved into the Mosquitto installation directory, e.g. 'C:\Program Files (x86)\mosquitto'.
    • Download 'libeay32.dll' here.
    • Download 'pthreadVC2.dll' here.
    • Download 'ssleay32.dll' here.
Finally we are able to start a local Mosquitto service on our computer:
  • Open the Windows Command Line
  • Navigate to the 'Mosquitto' installation directory.
  • Start the 'Mosquitto' service by typing "mosquitto" and press Enter.
  • If no error occurs a local MQTT broker will be available under 'tcp://localhost:1883'.

The Node-RED environment

How to get started
  • Download nodeJS from here.
  • After nodeJS has been successfully installed, open the command line:
    • cmd.exe
  • Please check the version of the installation now:
    • node --version && npm --version
    Something like the following should appear:
    v8.9.0
    5.5.1
  • Now install Node-RED from command line:
    • npm install -g --unsafe-perm node-red
  • Finally, after successful installation, please start Node-RED with the following command:
    • node-red

Getting Started with OPC UA

OPC UA Modeling and Client Software
  • In order to make use of OPC UA Modeling and to browse the respective UA Models, some software needs to be installed.
  • To develop information models, the "UaModeler" is needed. If not already installed, please download and install from here.
  • To browse OPC UA Servers, a Client application is needed. Please download and install the "UaExpert" from here.
Java OPC UA Client and Server SDK
  • Please download the Evaluation Package of the OPC UA Java implementation from here.
  • Unzip the Libraries from the archive.
  • Create a new project, e.g. "MyOpcUaServer".
  • Add the libraries to the Build Path of your project.
